# PLAN-Tools 测试执行指南

## 快速测试步骤

### 1. 打开应用
在浏览器中访问: http://localhost:5173

### 2. 打开浏览器控制台
按 F12 键打开开发者工具

### 3. 执行测试命令

依次执行以下命令:

#### 步骤1: 加载示例数据
```javascript
loadMockData()
```
预期输出: 📦 Mock data loaded successfully!

#### 步骤2: 查看数据统计
```javascript
getDataStats()
```
预期输出应包含:
- 项目名称: "示例项目 - 企业管理系统开发"
- 成员数: 6
- 任务总数: 18
- 已完成: 6
- 进行中: 2
- 待办: 10

#### 步骤3: 测试项目信息页面
1. 确认"项目信息管理"标签页激活
2. 检查项目名称字段显示
3. 检查开始日期和结束日期显示
4. 检查成员表格显示6个成员
5. 测试添加新成员:
   - 填写姓名: "测试人员"
   - 电话: "13900000000"
   - 邮箱: "test@example.com"
   - 角色: "开发工程师"
   - 点击"添加"按钮
6. 确认新成员添加成功

#### 步骤4: 测试项目计划页面
```javascript
// 导航到项目计划
document.querySelector('a[href="#/project-plan"]')?.click()
```
或者直接点击"项目计划管理"标签

预期结果:
1. 任务列表显示
2. 甘特图显示
3. 工具栏按钮可用
4. 任务数据正常显示

#### 步骤5: 测试任务操作
```javascript
// 点击新增按钮
document.querySelector('button:contains("新增")')?.click()
```

手动测试:
1. 点击"新增"按钮
2. 确认对话框打开
3. 填写任务信息:
   - 任务名称: "测试任务"
   - 开始日期: 选择日期
   - 结束日期: 选择日期
   - 优先级: "高"
   - 状态: "进行中"
4. 点击"保存"按钮
5. 确认任务添加成功

#### 步骤6: 测试显示设置
1. 点击"显示设置"按钮
2. 确认对话框打开
3. 切换一些列的显示状态
4. 点击"保存设置"
5. 确认任务列表列变化

#### 步骤7: 测试数据持久化
```javascript
// 重新加载页面
location.reload()

// 等待页面加载后检查数据
setTimeout(() => {
    const stats = getDataStats()
    console.log('任务总数:', stats.tasks.total)
    console.log('数据保持正常:', stats.tasks.total > 0)
}, 2000)
```

预期结果: 刷新后数据仍然存在

#### 步骤8: 测试导出功能
1. 点击"导出"按钮
2. 选择"JSON格式"
3. 确认文件下载

#### 步骤9: 测试甘特图导出
```javascript
// 查看甘特图是否存在
document.querySelector('.gantt-container')
```
预期结果: 元素存在

#### 步骤10: 测试响应式布局
```javascript
// 测试不同屏幕尺寸
const sizes = [
    [1920, 1080],  // 桌面
    [768, 1024],   // 平板
    [375, 667]     // 手机
]

sizes.forEach(([width, height]) => {
    window.resizeTo(width, height)
    console.log(`测试尺寸: ${width}x${height}`)
})
```

预期结果: 所有尺寸下页面正常显示

## 故障排除

### 问题1: 页面无法加载
**解决方案**: 检查开发服务器是否运行
```bash
npm run dev
```

### 问题2: 示例数据加载失败
**解决方案**:
1. 刷新页面
2. 重新执行 `loadMockData()`
3. 检查浏览器控制台是否有错误

### 问题3: 甘特图不显示
**解决方案**:
1. 刷新页面
2. 检查 dhtmlx-gantt 库是否加载
3. 查看控制台错误信息

### 问题4: 数据不保存
**解决方案**:
1. 检查 localStorage 是否启用
2. 清除浏览器缓存重试
3. 检查是否有浏览器扩展干扰
