## 一、技术栈选择

### 1.1 核心框架

**前端框架**：
- **Nuxt 3**: Vue 3全栈框架
  - SSR/SSG支持
  - 文件路由系统
  - 自动导入
  - Server Routes（API）
  - TypeScript支持

**构建工具**：
- **Nuxt CLI**: 项目脚手架和构建工具
- **Vite**: 底层构建引擎

### 1.2 状态管理

- **Pinia**: Vue 3官方状态管理
  - 项目信息状态（useProjectStore）
  - 任务数据状态（useTasksStore）
  - UI状态（useUIStore）
  - 认证状态（useAuthStore）⭐ Phase 2 — JWT认证 + Token刷新
  - 对话状态（useChatStore）⭐ Phase 2 — AI对话历史管理

### 1.3 UI组件库

- **Element Plus**: Vue 3组件库
- **Tailwind CSS**: 原子化CSS框架
- **dhtmlx-gantt**: 甘特图组件

### 1.4 国产大模型集成 ⭐

**DeepSeek（深度求索）**：
- **模型**: deepseek-chat（通用）
- **成本**: ¥1/100万tokens（输入）、¥2/100万tokens（输出）
- **优势**: 性价比极高、中文理解能力强、128K上下文
- **API格式**: OpenAI兼容格式（`/v1/chat/completions`）

> ⭐ **V0.4 说明**：当前版本统一使用 DeepSeek 作为 AI 服务商。架构设计上保留了切换其他 OpenAI 兼容格式服务商（如阿里QWen、智谱GLM）的能力，只需在 `providerConfig` 中增加配置项即可。

### 1.5 数据存储方案 ⭐

**Phase 1（MVP）**：
- **localStorage**: 用户项目数据临时存储
- **SessionStorage**: 会话数据
- **服务端环境变量**: AI API Key通过`.env`配置，服务端`useRuntimeConfig()`读取，前端不接触Key

> ⭐ **MVP阶段API Key安全方案**：API Key配置在服务端`.env`文件中，通过`nuxt.config.ts`的`runtimeConfig`暴露给Server Routes使用。前端只调用`/api/ai/*`接口，API Key始终在服务端，不暴露给浏览器。

**Phase 2（正式版）** ✅ 已完成：
- **PostgreSQL**: 主数据库（Drizzle ORM）
- **JWT**: 用户认证（Access Token + Refresh Token）
- **AES-256-CBC**: API Key加密存储

**Phase 3（商业化）**：
- **PostgreSQL + Redis**: 完整数据层
- **对象存储**: 文件存储（OSS/S3兼容）

### 1.6 部署方案 ⭐ NEW

**Docker自托管**：
- **Docker Compose**: 容器编排
- **Nginx**: 反向代理 + SSL
- **PostgreSQL**: 数据库容器
- **Redis**: 缓存容器

#### 2.2.1 📋 目录结构详解与迁移指导

**为什么从 .js 文件变成 .ts 文件？**

1. **Nuxt 3 默认支持 TypeScript**
   - Nuxt 3 内置 TypeScript 支持，提供最佳开发体验
   - 类型安全：减少运行时错误，提高代码质量
   - IDE 支持：更好的自动补全、重构和错误检测
   - Server Routes 必须使用 TypeScript（类型定义对于 API 接口至关重要）

2. **渐进式迁移策略** ⭐ **推荐方案**
   ```javascript
   // tsconfig.json 配置
   {
     "compilerOptions": {
       "allowJs": true,              // ✅ 允许 .js 文件存在
       "checkJs": false,             // ❌ 不检查 .js 文件类型
       "noEmit": true,
       "moduleResolution": "node"
     }
   }
   ```

   **迁移步骤**：
   ```
   阶段1 (MVP): 保持 .js 文件
     ├── store/project.js    ← 保持 JavaScript
     ├── store/tasks.js      ← 保持 JavaScript
     └── utils/export.js     ← 保持 JavaScript

   阶段2 (稳定后): 逐步迁移到 .ts
     ├── store/project.ts    ← 添加类型
     ├── store/tasks.ts      ← 添加类型
   ```

**为什么从 views/ 变成 pages/？**

| 特性 | Vue 3 (views/) | Nuxt 3 (pages/) |
|------|----------------|-----------------|
| **路由方式** | 编程式路由 (手动配置) | 文件路由 (自动生成) |
| **配置文件** | src/router/index.js | 无需配置 |
| **路由生成** | 需要手动定义每个路由 | 根据文件自动生成 |
| **动态路由** | 需要 path: '/user/:id' | 文件名: [id].vue |
| **嵌套路由** | 需要 children 配置 | 文件夹结构自动嵌套 |

### 3.1 AI服务封装

**统一接口设计**：

> ⭐ **V0.4 说明**：当前版本统一使用 DeepSeek 作为 AI 服务商，通过 OpenAI 兼容格式（`/v1/chat/completions`）调用。架构设计上保留了切换其他 OpenAI 兼容格式服务商的能力。

```typescript
// server/utils/ai/types.ts
interface Message {
  role: 'system' | 'user' | 'assistant'
  content: string
}

interface AIResponse {
  content: string
  model: string
  provider: string
  usage: {
    promptTokens: number
    completionTokens: number
    totalTokens: number
  }
}

// server/utils/ai/client.ts
export async function callAI(
  messages: Message[],
  options?: { model?: string; temperature?: number; maxTokens?: number }
): Promise<AIResponse> {
  const config = useRuntimeConfig()

  const baseURL = config.deepseekApiBase || 'https://api.deepseek.com'
  const apiKey = config.deepseekApiKey
  const model = options?.model || config.deepseekModel || 'deepseek-chat'

  if (!apiKey) {
    throw createError({
      statusCode: 500,
      statusMessage: 'Deepseek API Key not configured. Please set it in .env file.'
    })
  }

  try {
    const response = await $fetch('/chat/completions', {
      baseURL,
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json'
      },
      body: {
        model,
        messages,
        temperature: options?.temperature ?? 0.7,
        max_tokens: options?.maxTokens ?? 2000
      }
    })

    return {
      content: response.choices[0]?.message?.content || '',
      model,
      provider: 'deepseek',
      usage: {
        promptTokens: response.usage?.prompt_tokens || 0,
        completionTokens: response.usage?.completion_tokens || 0,
        totalTokens: response.usage?.total_tokens || 0
      }
    }
  } catch (error: any) {
    console.error('AI API error (deepseek):', error?.message || error)
    throw createError({
      statusCode: error?.statusCode || 500,
      statusMessage: `Failed to call Deepseek API: ${error?.message || 'Unknown error'}`
    })
  }
}
```

### 3.4 错误处理与重试

```typescript
// server/utils/ai/retry.ts
async function callAIWithRetry(
  fn: () => Promise<any>,
  maxRetries = 3
) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn()
    } catch (error: any) {
      if (i === maxRetries - 1) throw error

      if (error?.statusCode === 429) {
        await new Promise(resolve => setTimeout(resolve, 1000 * (i + 1)))
      } else if (error?.statusCode >= 500) {
        continue
      } else {
        throw error
      }
    }
  }
}
```

## 十六、开发规范

### 16.1 代码规范

**TypeScript规范**：
- 使用严格模式
- 所有函数添加类型注解
- 使用接口定义数据结构
- 避免使用any类型

**Vue组件规范**：
- 使用Composition API + `<script setup>`
- 组件文件名使用PascalCase
- 组件目录使用PascalCase
- 组合式函数使用camelCase

**API接口规范**：
- RESTful API设计
- 使用HTTP动词（GET/POST/PUT/DELETE）
- 资源名使用复数形式
- 统一响应格式
