## 一、技术栈选择

### 1.1 核心框架

**前端框架**：
- **Nuxt 3**: Vue 3全栈框架
  - SSR/SSG支持
  - 文件路由系统
  - 自动导入
  - Server Routes（API）
  - TypeScript支持

**构建工具**：
- **Nuxt CLI**: 项目脚手架和构建工具
- **Vite**: 底层构建引擎

### 1.2 状态管理

- **Pinia**: Vue 3官方状态管理
  - 项目信息状态（useProjectStore）
  - 任务数据状态（useTasksStore）
  - UI状态（useUIStore）
  - AI配置状态（useAIStore）⭐ Phase 2 — MVP阶段API Key在服务端管理，无需前端Store

### 1.3 UI组件库

- **Element Plus**: Vue 3组件库
- **Tailwind CSS**: 原子化CSS框架
- **dhtmlx-gantt**: 甘特图组件

### 1.4 国产大模型集成 ⭐ NEW

**DeepSeek（深度求索）**：
- **模型**: deepseek-chat（通用）、deepseek-coder（代码）
- **成本**: ¥1/100万tokens（输入）、¥2/100万tokens（输出）
- **优势**: 性价比极高、中文理解能力强、128K上下文

**阿里QWen（通义千问）**：
- **模型**: qwen-max（旗舰）、qwen-plus（性价比）、qwen-turbo（快速）
- **成本**: ¥0.8-120/100万tokens
- **优势**: 企业级稳定性、阿里生态集成、Function Calling
- **API格式**: OpenAI兼容格式（`/compatible-mode/v1/chat/completions`），与DeepSeek共用同一接口结构

> ⭐ **重要**：QWen已全面切换到OpenAI兼容格式，两个模型可共用一套客户端代码，只需切换`baseURL`和`apiKey`，大幅简化开发。

### 1.5 数据存储方案 ⭐

**Phase 1（MVP）**：
- **localStorage**: 用户项目数据临时存储
- **SessionStorage**: 会话数据
- **服务端环境变量**: AI API Key通过`.env`配置，服务端`useRuntimeConfig()`读取，前端不接触Key

> ⭐ **MVP阶段API Key安全方案**：API Key配置在服务端`.env`文件中，通过`nuxt.config.ts`的`runtimeConfig`暴露给Server Routes使用。前端只调用`/api/ai/*`接口，API Key始终在服务端，不暴露给浏览器。用户如需切换Key，修改服务端`.env`后重启应用即可。

**Phase 2（正式版）**：
- **PostgreSQL**: 主数据库（通过Supabase或自托管）
- **Redis**: 缓存层（可选）

**Phase 3（商业化）**：
- **PostgreSQL + Redis**: 完整数据层
- **对象存储**: 文件存储（OSS/S3兼容）

### 1.6 部署方案 ⭐ NEW

**Docker自托管**：
- **Docker Compose**: 容器编排
- **Nginx**: 反向代理 + SSL
- **PostgreSQL**: 数据库容器
- **Redis**: 缓存容器

#### 2.2.1 📋 目录结构详解与迁移指导

**为什么从 .js 文件变成 .ts 文件？**

1. **Nuxt 3 默认支持 TypeScript**
   - Nuxt 3 内置 TypeScript 支持，提供最佳开发体验
   - 类型安全：减少运行时错误，提高代码质量
   - IDE 支持：更好的自动补全、重构和错误检测
   - Server Routes 必须使用 TypeScript（类型定义对于 API 接口至关重要）

2. **渐进式迁移策略** ⭐ **推荐方案**
   ```javascript
   // tsconfig.json 配置
   {
     "compilerOptions": {
       "allowJs": true,              // ✅ 允许 .js 文件存在
       "checkJs": false,             // ❌ 不检查 .js 文件类型
       "noEmit": true,
       "moduleResolution": "node"
     }
   }
   ```

   **迁移步骤**：
   ```
   阶段1 (MVP): 保持 .js 文件
     ├── store/project.js    ← 保持 JavaScript
     ├── store/tasks.js      ← 保持 JavaScript
     └── utils/export.js     ← 保持 JavaScript

   阶段2 (稳定后): 逐步迁移到 .ts
     ├── store/project.ts    ← 添加类型
     ├── store/tasks.ts      ← 添加类型
     └── utils/export.ts     ← 添加类型

   阶段3 (完成): 全面类型化
     ├── types/              ← 完整类型定义
     ├── store/*.ts         ← 完整类型支持
     └── utils/*.ts         ← 完整类型支持
   ```

3. **Vue 组件不需要立即改名**
   - `.vue` 文件可以继续使用 `<script setup lang="js">`
   - 或者逐步迁移到 `<script setup lang="ts">`
   - 两种语法可以共存

**为什么从 views/ 变成 pages/？**

| 特性 | Vue 3 (views/) | Nuxt 3 (pages/) |
|------|----------------|-----------------|
| **路由方式** | 编程式路由 (手动配置) | 文件路由 (自动生成) |
| **配置文件** | src/router/index.js | 无需配置 |
| **路由生成** | 需要手动定义每个路由 | 根据文件自动生成 |
| **动态路由** | 需要 path: '/user/:id' | 文件名: [id].vue |
| **嵌套路由** | 需要 children 配置 | 文件夹结构自动嵌套 |

**对比示例**：
```javascript
// ❌ Vue 3 方式 (src/router/index.js)
const routes = [
  { path: '/project-info', component: () => import('@/views/ProjectInfoView.vue') },
  { path: '/project-plan', component: () => import('@/views/ProjectPlanView.vue') },
  { path: '/projects/:id', component: () => import('@/views/ProjectDetail.vue') }
]

// ✅ Nuxt 3 方式 (pages 目录)
// src/pages/
//   ├── index.vue              → /
//   ├── project-info.vue       → /project-info
//   ├── project-plan.vue       → /project-plan
//   └── projects/
//       └── [id].vue           → /projects/:id
```

**main.js 如何迁移？**

Vue 3 的 `src/main.js` 在 Nuxt 3 中被拆分为多个文件：

```javascript
// ❌ Vue 3: src/main.js
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import ElementPlus from 'element-plus'
import { createPinia } from 'pinia'

const app = createApp(App)
app.use(router)
app.use(ElementPlus)
app.use(createPinia())
app.mount('#app')

// ✅ Nuxt 3: 拆分为以下文件
```

**1. app.vue (根组件)**
```vue
<!-- src/app.vue -->
<template>
  <NuxtLayout>
    <NuxtPage />
  </NuxtLayout>
</template>
```

**2. i18n配置（通过 @nuxtjs/i18n 模块管理）**

> ⭐ **重要**：不要手动创建 `plugins/i18n.ts`。使用 `@nuxtjs/i18n` 模块，在 `nuxt.config.ts` 中配置即可。手动创建插件会与模块冲突导致重复初始化。
>
> 安装：`pnpm add @nuxtjs/i18n`
>
> 配置见下方 `nuxt.config.ts` 中的 `i18n` 选项。

> ⭐ **注意**：`@pinia/nuxt`和`@element-plus/nuxt`作为Nuxt模块已在`nuxt.config.ts`的`modules`中注册，**不需要**手动创建`plugins/pinia.ts`和`plugins/element-plus.ts`，否则会导致重复初始化。

**3. nuxt.config.ts (Nuxt 主配置)**
```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  srcDir: 'src/',

  modules: [
    '@pinia/nuxt',
    '@element-plus/nuxt',
    '@nuxtjs/tailwindcss',
    '@nuxtjs/i18n'
  ],

  css: ['~/assets/main.css'],

  app: {
    pageTransition: { name: 'page', mode: 'out-in' }
  },

  elementPlus: {
    locale: 'zh-cn'
  },

  i18n: {
    locales: [
      { code: 'zh-CN', name: '中文', file: 'zh-CN.json' },
      { code: 'en', name: 'English', file: 'en.json' }
    ],
    defaultLocale: 'zh-CN',
    fallbackLocale: 'en',
    langDir: 'locales/',
    lazy: true
  },

  runtimeConfig: {
    deepseekApiKey: process.env.DEEPSEEK_API_KEY || '',
    deepseekApiBase: process.env.DEEPSEEK_API_BASE || 'https://api.deepseek.com',
    deepseekModel: process.env.DEEPSEEK_MODEL || 'deepseek-chat',
    qwenApiKey: process.env.QWEN_API_KEY || '',
    qwenApiBase: process.env.QWEN_API_BASE || 'https://dashscope.aliyuncs.com/compatible-mode/v1',
    qwenModel: process.env.QWEN_MODEL || 'qwen-max',
  }
})
```

#### 2.2.2 🔄 Vue 3 到 Nuxt 3 文件映射表

| - | `server/api/` | ✨ **新增** | Server Routes (API) |
| - | `server/utils/` | ✨ **新增** | 服务端工具函数 |

#### 2.2.3 🛠️ 迁移步骤详解

**步骤 1: 创建 Nuxt 3 项目** (保留原项目)
```bash
# 在新目录创建 Nuxt 3 项目
npx nuxi@latest init plan-tools-nuxt

# 或者手动迁移到当前项目
npm install nuxt@3 @nuxt/kit
```

**步骤 2: 复制前端代码**
```bash
# 复制组件 (可直接使用)
cp -r src/components plan-tools-nuxt/src/

# 复制工具函数 (保持 .js，允许JS/TS共存)
cp -r src/utils plan-tools-nuxt/src/
cp -r src/store plan-tools-nuxt/src/
cp -r src/locales plan-tools-nuxt/src/
cp -r src/data plan-tools-nuxt/src/
cp -r src/assets plan-tools-nuxt/src/
```

**步骤 3: 移动视图到 pages**
```bash
# 重命名并移动 views → pages
mv src/views/* plan-tools-nuxt/pages/
```

**步骤 4: 拆分 main.js**
```bash
# 创建插件目录
mkdir -p plan-tools-nuxt/plugins

# 拆分为多个文件 (参考上方示例)
# - app.vue
# - nuxt.config.ts (含 modules 注册 Pinia/Element Plus/i18n)
# - locales/zh-CN.json, locales/en.json
```

**步骤 5: 配置 TypeScript 支持 JS**
```typescript
// tsconfig.json
{
```

## 三、AI集成架构 ⭐ NEW

### 3.1 AI服务封装

**统一接口设计**：

> ⭐ **重要**：DeepSeek和QWen均支持OpenAI兼容格式（`/v1/chat/completions`），因此可以共用一套客户端代码，只需切换`baseURL`和`apiKey`。

```typescript
// server/utils/ai/types.ts
interface Message {
  role: 'system' | 'user' | 'assistant'
  content: string
}

interface AIResponse {
  content: string
  model: string
  provider: string
  usage: {
    promptTokens: number
    completionTokens: number
    totalTokens: number
  }
}

// server/utils/ai/client.ts
export async function callAI(
  messages: Message[],
  options?: { provider?: 'deepseek' | 'qwen'; model?: string; temperature?: number }
): Promise<AIResponse> {
  const config = useRuntimeConfig()

  const provider = options?.provider || 'deepseek'

  const providerConfig = {
    deepseek: {
      baseURL: config.deepseekApiBase || 'https://api.deepseek.com',
      apiKey: config.deepseekApiKey,
      model: options?.model || config.deepseekModel || 'deepseek-chat'
    },
    qwen: {
      baseURL: config.qwenApiBase || 'https://dashscope.aliyuncs.com/compatible-mode/v1',
      apiKey: config.qwenApiKey,
      model: options?.model || config.qwenModel || 'qwen-max'
    }
  }

  const { baseURL, apiKey, model } = providerConfig[provider]

  if (!apiKey) {
    throw createError({
      statusCode: 500,
      statusMessage: `${provider} API Key not configured. Please set it in .env file.`
    })
  }

  try {
    const response = await $fetch<{ choices: Array<{ message: { content: string } }>; usage: { prompt_tokens: number; completion_tokens: number; total_tokens: number } }>(
      `${baseURL}/chat/completions`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${apiKey}`,
          'Content-Type': 'application/json'
        },
        body: {
          model,
          messages,
          temperature: options?.temperature ?? 0.7,
          max_tokens: 2000
        }
      }
    )

    return {
      content: response.choices[0]?.message?.content || '',
      model,
      provider,
      usage: {
        promptTokens: response.usage?.prompt_tokens || 0,
        completionTokens: response.usage?.completion_tokens || 0,
        totalTokens: response.usage?.total_tokens || 0
      }
    }
  } catch (error: any) {
    console.error(`AI API error (${provider}):`, error?.message || error)
    throw createError({
      statusCode: error?.statusCode || 500,
      statusMessage: `Failed to call ${provider} API: ${error?.message || 'Unknown error'}`
    })
  }
}
```

> ⭐ **架构优势**：统一客户端意味着添加新的AI服务商（如智谱GLM、百度文心）只需在`providerConfig`中增加一个配置项，无需编写新的Provider类。

### 3.4 错误处理与重试

```typescript
// server/utils/ai/retry.ts
async function callAIWithRetry(
  fn: () => Promise<any>,
  maxRetries = 3
) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn()
    } catch (error: any) {
      if (i === maxRetries - 1) throw error

      if (error?.statusCode === 429) {
        await new Promise(resolve => setTimeout(resolve, 1000 * (i + 1)))
      } else if (error?.statusCode >= 500) {
        continue
      } else {
        throw error
      }
    }
  }
}
```

### 10.1 API Key安全

**MVP阶段（服务端环境变量）**：

> ⭐ **核心原则**：API Key永远不离开服务端。前端只调用`/api/ai/*`接口，由服务端代理AI请求。

```typescript
})

// server/api/ai/chat.post.ts - 服务端使用

  const config = useRuntimeConfig()

  // ... 使用apiKey调用AI
```

**Phase 2+（用户级API Key，数据库加密存储）**：
```typescript
import { createCipheriv, randomBytes, createDecipheriv } from 'crypto'

function encryptApiKey(key: string): string {
  const iv = randomBytes(16)
  const cipher = createCipheriv('aes-256-cbc', Buffer.from(ENCRYPTION_KEY), iv)
  let encrypted = cipher.update(key, 'utf8', 'hex')
  encrypted += cipher.final('hex')
  return iv.toString('hex') + ':' + encrypted
}

function decryptApiKey(encryptedKey: string): string {
  const parts = encryptedKey.split(':')
  const iv = Buffer.from(parts[0], 'hex')
  const encrypted = parts[1]
  const decipher = createDecipheriv('aes-256-cbc', Buffer.from(ENCRYPTION_KEY), iv)
  let decrypted = decipher.update(encrypted, 'hex', 'utf8')
  decrypted += decipher.final('utf8')
  return decrypted
}
```

> ⭐ **安全要点**：
> - `runtimeConfig`中的非`public`前缀字段只在服务端可用，不会序列化到前端
> - 如需前端可访问的配置，必须使用`public`前缀：`runtimeConfig.public.xxx`
> - **绝不要**将API Key放在`runtimeConfig.public`中

### 10.2 速率限制

```typescript
// server/middleware/rate-limit.ts
import { defineEventHandler } from 'h3'

const rateLimit = new Map()

export default defineEventHandler(async (event) => {
  const ip = event.node.req.headers['x-forwarded-for'] || event.node.req.socket.remoteAddress

  const requests = rateLimit.get(ip) || []
  const now = Date.now()
  const recentRequests = requests.filter(t => now - t < 60000) // 最近1分钟

  if (recentRequests.length >= 100) {
    throw createError({
      statusCode: 429,
      statusMessage: 'Too Many Requests'
    })
  }

  recentRequests.push(now)
  rateLimit.set(ip, recentRequests)
})
```
