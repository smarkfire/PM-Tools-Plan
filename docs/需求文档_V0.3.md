## 产品概述

### 产品名称
PLAN-Tools（项目计划管理软件）

### 产品定位
一个基于Nuxt 3全栈架构的项目管理应用，通过集成AI能力，帮助用户快速创建和管理专业项目计划。

### 目标用户
- 项目管理新手：需要快速上手项目规划
- 小型团队：需要简单的任务管理工具
- 独立开发者：需要个人项目管理

### 核心价值
- **低门槛**：AI辅助快速创建专业项目计划
- **高效率**：自动化重复性工作
- **易使用**：简洁直观的界面设计
- **全栈能力**：服务端AI调用保证安全性

### 一、项目信息管理

**功能描述**：管理项目的基本信息

**信息字段**：
- 项目名称（必填）
- 项目开始日期（必填）
- 项目结束日期（必填）
- 项目备注（选填）

**操作功能**：
- 编辑项目信息
- 保存到本地存储/数据库

#### 1.2 项目人员管理
**功能描述**：管理项目团队成员信息

**人员信息字段**：
- 姓名（必填）

**操作功能**：
- 新增人员
- 编辑人员
- 删除人员

#### 1.3 数据导入导出
**功能描述**：项目信息的导入导出

**支持格式**：
- JSON格式
- Excel格式

**操作功能**：
- 导入项目信息（覆盖当前数据）

- 删除任务

**操作功能**：

- 缩放时间轴

#### 2.4 数据导入导出
**功能描述**：项目计划的导入导出

**支持格式**：
- JSON格式

- Excel格式
- PNG格式（甘特图）

**操作功能**：
- 导入项目计划（支持追加/覆盖）
- 导出项目计划
- 导出甘特图为PNG图片

#### 2.5 显示设置
**功能描述**：自定义任务列表显示的字段

**可选字段**：
- WBS编号
- 任务名称

- 结束时间

### 三、AI智能规划 ⭐ NEW

#### 3.1 AI配置管理
**功能描述**：配置AI服务的连接信息（服务端管理）

**配置项**：
- AI服务提供商选择（DeepSeek / 阿里QWen）
- 模型选择（deepseek-chat / qwen-max等）
- API端点配置（支持私有化部署）
- 配置验证

> ⭐ **MVP阶段API Key安全方案**：API Key配置在服务端`.env`文件中，通过`nuxt.config.ts`的`runtimeConfig`暴露给Server Routes使用。前端只调用`/api/ai/*`接口，API Key始终在服务端，不暴露给浏览器。用户如需切换Key，修改服务端`.env`后重启应用即可。

**数据存储**：
- **Phase 1（MVP）**：服务端环境变量（`.env`文件），通过`runtimeConfig`读取，前端不接触Key
- **Phase 2（正式版）**：服务端数据库加密存储，支持用户级API Key管理
- **Phase 3（商业化）**：平台统一管理，按使用量计费

**安全性**：
- API Key在服务端管理，不暴露给前端
- 前端只接收AI响应结果
- `runtimeConfig`中非`public`前缀字段只在服务端可用，不会序列化到前端
- 支持API Key轮换机制（修改`.env`后重启）
- 支持企业内网私有化部署

#### 3.2 智能项目规划
**功能描述**：通过AI自动生成项目任务计划

**输入**：
- 项目描述（文本输入）
- 行业选择（软件开发、市场营销、建筑施工等）
- 特殊要求（选填）

**AI生成内容**：
- 任务列表（多层级结构）
- 任务工期估算
- 任务依赖关系
- 任务描述和验收标准
- WBS编号

**操作流程**：
1. 点击"AI智能规划"按钮
2. 输入项目描述
3. 选择行业模板（可选）
4. 后端调用DeepSeek/QWen生成任务列表
5. 预览和编辑生成的任务
6. 确认导入到项目

**展示功能**：
- 任务列表预览（树形结构）
- 甘特图预览
- 统计信息（任务总数、总工期、关键路径任务数）
- 任务编辑（支持在线修改）
- 关键路径高亮
- 循环依赖警告

#### 3.3 行业模板库
**功能描述**：提供预设的行业项目模板

**内置模板**：
- 软件开发模板（约25个任务节点）
  - 需求分析阶段
  - 架构设计阶段
  - 编码开发阶段
  - 测试阶段
  - 上线部署阶段

- 市场营销模板（约20个任务节点）
  - 市场调研阶段
  - 策划阶段
  - 执行阶段
  - 复盘阶段

- 建筑施工模板（约22个任务节点）
  - 基础工程阶段
  - 主体工程阶段
  - 装修工程阶段
  - 验收交付阶段

**模板功能**：
- 模板预览（显示阶段和任务列表）
- 一键应用模板
- 模板自定义（保存为个人模板）
- **Phase 2+**：模板共享市场

#### 3.4 AI工期估算
**功能描述**：通过AI估算任务工期

**估算依据**：
- 任务描述
- 任务复杂度（1-5级）
- 团队成员能力
- 历史数据参考（**Phase 2+**）

**输出结果**：
- 乐观工期（最快完成时间）
- 最可能工期
- 悲观工期（最慢完成时间）
- 期望工期（PERT公式计算）
- 置信度（0-100%）

**使用场景**：
- 创建任务时自动估算
- 手动触发估算
- 批量估算

#### 3.5 依赖关系推荐
**功能描述**：AI分析并推荐任务间的依赖关系

**推荐逻辑**：
- 识别任务间的逻辑关系
- 区分串行/并行关系
- 避免循环依赖
- 标注关键路径

**展示功能**：
- 可视化依赖关系
- 推荐理由说明
- 一键应用推荐
- 手动调整依赖

### 四、AI问答助手 ⭐ NEW

#### 4.1 助手入口
**功能描述**：提供AI助手的访问入口

**入口形式**：
- 页面右下角浮动按钮
- AI助手图标
- 未读消息红点提示
- Hover显示"AI助手"提示

#### 4.2 对话界面
**功能描述**：与AI助手进行对话交互

**界面布局**：
- 标题栏（AI助手 + 最小化/关闭按钮）
- 消息区域
  - 用户消息（右侧气泡）
  - AI消息（左侧气泡）
- 输入区域（文本输入框 + 发送按钮）
- 欢迎语（展示助手能力）

**交互功能**：
- 发送文本消息
- 查看对话历史（**Phase 1**：本地100条，**Phase 2+**：数据库无限）
- 清空对话
- 复制消息内容
- 流式响应（**Phase 2+**）

#### 4.3 项目状态查询
**功能描述**：通过自然语言查询项目状态

**支持的查询类型**：

**延期风险查询**：
- 示例："哪些任务可能延期？"
- 返回：延期风险任务列表、延期天数、风险等级

**成员负载查询**：
- 示例："张三下周的工作负载如何？"
- 返回：成员任务数、工作负载百分比、负载状态（正常/过载）

**项目健康查询**：
- 示例："项目健康度如何？"
- 返回：健康评分（0-100）、进度分析、改进建议

**依赖关系查询**：
- 示例："任务A的前置任务是什么？"
- 返回：前置/后置任务列表、完整依赖链

**What-if分析**：
- 示例："如果任务A延期3天，会影响哪些任务？"
- 返回：影响范围分析、关键路径变化

#### 4.4 操作执行
**功能描述**：通过自然语言执行项目操作

**支持的操作类型**：
- 分配任务："把任务A分配给张三"
- 修改工期："把任务A的工期改为5天"
- 修改状态："把任务A标记为已完成"
- 添加备注："给任务A添加备注..."
- 设置优先级："把任务A优先级设为高"

**操作流程**：
1. 用户输入操作指令
2. 后端AI解析操作意图和参数
3. 弹出确认对话框（显示操作内容）
4. 用户确认后通过API执行
5. 显示执行结果

#### 4.5 项目健康诊断
**功能描述**：AI综合评估项目健康状态

**诊断维度**：
- 进度健康（实际进度 vs 计划进度）
- 资源健康（成员负载分布）
- 风险健康（延期风险任务数）
- 依赖健康（关键路径状态）

**输出内容**：
- 综合健康评分（0-100分）
- 各维度详细分析
- 可视化图表展示
- 改进建议列表

#### 4.6 自动文档生成
**功能描述**：AI自动生成项目管理文档

**支持文档类型**：

**项目周报**：
- 本周进展（完成任务列表）
- 下周计划（待开展任务）
- 风险提示（延期风险）
- 需协调事项

**项目月报**：
- 月度总结
- 关键指标统计
- 里程碑完成情况

**里程碑总结**：
- 成果回顾
- 问题总结
- 经验教训

**项目复盘报告**：
- 项目整体评估
- 成功经验
- 改进建议

**操作流程**：
1. 选择文档类型
2. 后端AI生成文档内容
3. 预览和编辑
4. 导出（支持多种格式）

### 六、数据持久化 ⭐

#### 6.1 存储方案演进

**Phase 1：MVP阶段（当前）**
```
技术栈：
- 前端：Nuxt 3 + Vue 3
- 后端：Nuxt Server Routes
- AI调用：服务端API
- 存储：localStorage

存储位置：
- 项目信息：localStorage (plan-tools-project)
- 任务数据：localStorage (plan-tools-tasks)
- UI状态：localStorage (plan-tools-ui)
- AI配置：localStorage (plan-tools-ai)
- 对话历史：localStorage (plan-tools-chat，限制100条)

优点：
✅ 快速启动，无需数据库
✅ 开发简单，成本低
✅ 适合单用户场景

限制：
❌ 无法多用户协作
❌ 数据不安全（本地存储）
❌ 无法做AI调用统计
```

**Phase 2：正式产品**
```
技术栈：
- 前端：Nuxt 3 + Vue 3
- 后端：Nuxt Server Routes
- 数据库：PostgreSQL (Supabase)
- 缓存：Redis（可选）
- AI调用：服务端API

存储位置：
- 用户数据：PostgreSQL users表
- 项目数据：PostgreSQL projects表
- 任务数据：PostgreSQL tasks表
- AI配置：PostgreSQL ai_configs表（加密）
- 对话历史：PostgreSQL chat_history表
- 行业模板：PostgreSQL templates表

新增功能：
✅ 用户认证和授权
✅ 多用户协作
✅ 数据云端同步
✅ AI调用统计（配额管理）
✅ 模板市场
```

**Phase 3：商业化**
```
技术栈：
- 前端：Nuxt 3 + Vue 3
- 后端：Nuxt Server Routes
- 数据库：PostgreSQL + Redis
- 监控：Sentry（错误追踪）
- 分析：Google Analytics

新增功能：
✅ AI Token计费
✅ 多租户支持
✅ 企业版功能
✅ API速率限制
✅ 高级分析报表
```

│                                                              │

```

### Nuxt 3 + 自托管优势

| 优势 | 说明 |
|------|------|
| **统一开发体验** | 前端后端同一仓库，同一语言（TypeScript） |
| **API Key 安全** | 敏感逻辑在服务端，API Key不暴露前端 |
| **数据自主可控** | 数据存储在自己服务器，符合企业安全要求 |
| **国产大模型** | 使用DeepSeek/阿里QWen，成本更低，响应更快 |
| **支持私有化** | 支持企业内网私有化部署 |
| **SSR/SSG 支持** | SEO友好，首屏加载快 |
| **类型安全** | 全栈TypeScript，类型自动推导 |
| **部署灵活** | 支持Docker、K8s等多种部署方式 |

## API接口设计

### AI相关接口

#### 1. AI对话接口
```
POST /api/ai/chat

Request:
{
  "messages": [
    { "role": "user", "content": "哪些任务可能延期？" }
  ],
  "projectContext": {
    "projectId": "xxx",
    "taskCount": 23,
    "memberCount": 5
  }
}

Response:
{
  "content": "根据分析，以下任务存在延期风险...",
  "usage": {
    "promptTokens": 150,
    "completionTokens": 200,
    "totalTokens": 350
  }
}
```

#### 2. WBS生成接口
```
POST /api/ai/wbs

Request:
{
  "projectDescription": "开发一个电商平台",
  "industry": "软件开发",
  "requirements": "包含商品管理、购物车、支付功能"
}

Response:
{
  "tasks": [
    {
      "id": "task-1",
      "name": "需求分析",
      "duration": 5,
      "description": "...",
      "children": [...]
    }
  ],
  "statistics": {
    "totalTasks": 23,
    "estimatedDuration": 85,
    "criticalPathCount": 6
  }
}
```

#### 3. 工期估算接口
```
POST /api/ai/estimate-duration

Request:
{
  "taskDescription": "开发用户登录功能",
  "complexity": 3,
  "teamSkills": ["Vue.js", "Node.js"]
}

Response:
{
  "optimistic": 3,      // 乐观工期（天）
  "likely": 5,          // 最可能工期
  "pessimistic": 8,     // 悲观工期
  "expected": 5.17,     // 期望工期（PERT）
  "confidence": 0.85    // 置信度
}
```

#### 4. 依赖关系推荐接口
```
POST /api/ai/suggest-dependencies

Request:
{
  "tasks": [
    { "id": "1", "name": "需求分析" },
    { "id": "2", "name": "UI设计" },
    { "id": "3", "name": "前端开发" }
  ]
}

Response:
{
  "dependencies": [
    {
      "taskId": "2",
      "dependsOn": ["1"],
      "reason": "UI设计需要基于需求分析的结果"
    },
    {
      "taskId": "3",
      "dependsOn": ["2"],
      "reason": "前端开发需要UI设计稿"
    }
  ]
}
```

#### 5. 项目健康诊断接口
```
POST /api/ai/health-diagnosis

Request:
{
  "projectId": "xxx"
}

Response:
{

- `qwen-plus` - 性价比模型

**优势**：

**API格式**：OpenAI兼容格式（`/compatible-mode/v1/chat/completions`）

> ⭐ **重要**：DeepSeek和QWen均支持OpenAI兼容格式，因此使用统一客户端`callAI()`，只需切换`baseURL`和`apiKey`，无需为每个服务商创建单独的接口文件。

**统一AI客户端**：
```typescript
// server/utils/ai/client.ts
export async function callAI(
  messages: Message[],
  options?: { provider?: 'deepseek' | 'qwen'; model?: string; temperature?: number }
): Promise<AIResponse> {
  const config = useRuntimeConfig()

  const provider = options?.provider || 'deepseek'

  const providerConfig = {
    deepseek: {
      baseURL: config.deepseekApiBase || 'https://api.deepseek.com',
      apiKey: config.deepseekApiKey,
      model: options?.model || config.deepseekModel || 'deepseek-chat'
    },
    qwen: {
      baseURL: config.qwenApiBase || 'https://dashscope.aliyuncs.com/compatible-mode/v1',
      apiKey: config.qwenApiKey,
      model: options?.model || config.qwenModel || 'qwen-max'
    }
  }

  const { baseURL, apiKey, model } = providerConfig[provider]

  if (!apiKey) {
    throw createError({
      statusCode: 500,
      statusMessage: `${provider} API Key not configured. Please set it in .env file.`
    })
  }

  const response = await $fetch(`${baseURL}/chat/completions`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    },
    body: {
      model,
      messages,
      temperature: options?.temperature ?? 0.7,
      max_tokens: 2000
    }
  })

  return {
    content: response.choices[0]?.message?.content || '',
    model,
    provider,
    usage: {
      promptTokens: response.usage?.prompt_tokens || 0,
      completionTokens: response.usage?.completion_tokens || 0,
      totalTokens: response.usage?.total_tokens || 0
    }
  }
}
```

> ⭐ **架构优势**：统一客户端意味着添加新的AI服务商（如智谱GLM、百度文心）只需在`providerConfig`中增加一个配置项，无需编写新的Provider类。

### 模型切换策略

**自动选择机制**：
```typescript
// server/utils/ai/router.ts
type AITaskType = 'task-breakdown' | 'duration-estimation' | 'chat' | 'health-diagnosis' | 'report-generation'

interface AIProviderConfig {
  provider: 'deepseek' | 'qwen'
  model: string
}

export function selectAIProvider(taskType: AITaskType): AIProviderConfig {
  const modelConfig: Record<AITaskType, AIProviderConfig> = {
    // 任务分解：使用DeepSeek（便宜且准确）
    'task-breakdown': {
      provider: 'deepseek',
      model: 'deepseek-chat'
    },

    // 工期估算：使用Qwen Max（更精确）
    'duration-estimation': {
      provider: 'qwen',
      model: 'qwen-max'
    },

    // 问答对话：使用Qwen Turbo（快速响应）
    'chat': {
      provider: 'qwen',
      model: 'qwen-turbo'
    },

    // 健康诊断：使用DeepSeek
    'health-diagnosis': {
      provider: 'deepseek',
      model: 'deepseek-chat'
    },

    // 文档生成：使用Qwen Plus
    'report-generation': {
      provider: 'qwen',
      model: 'qwen-plus'
    }
  }

  return modelConfig[taskType]
}
```

## 安全设计

### API Key管理

**Phase 1（MVP）**：
```
用户自带API Key模式：
1. 用户在设置中输入自己的API Key
2. 加密存储在localStorage
3. 每次请求时发送到后端
4. 后端使用该Key调用AI API
5. 用完后立即清除（不记录日志）
```

**Phase 2+（正式版）**：
```
平台统一管理模式：
1. 平台统一购买API配额（DeepSeek/QWen）
2. 用户注册后获得免费额度
3. API Key存储在数据库加密字段
4. 服务端统一调用和计费
5. 超额后引导用户充值
```

**企业私有化部署**：
```
企业内网模式：
1. 企业自行部署DeepSeek/QWen私有化版本
2. API Key存储在企业密钥管理系统
3. 所有数据不出内网
4. 完全自主可控
```
