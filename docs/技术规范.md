# PLAN-Tools 技术规范

## 1. 技术栈选择

### 1.1 前端框架

- **框架**: Vue 3
- **构建工具**: Vite
- **样式方案**: Tailwind CSS + Element Plus
- **状态管理**: Pinia
- **路由管理**: Vue Router&#x20;
- **UI库**: Element Plus

### 1.2 第三方库

- **甘特图**:  dhtmlx-gantt
- **表格处理**: xlsx (用于Excel导入导出)
- **日期处理**: dayjs
- **图标库**: Font Awesome
- **拖拽功能**: Sortable.js (用于任务排序)
- **UI组件库**: Element Plus&#x20;
- **代码规范**: ESLint + Prettier
- **单元测试**: Vitest
- **端到端测试**: Cypress

### 2.2 数据流设计

- **单向数据流**: 状态管理采用Pinia，确保数据流的可预测性
- **本地存储**: 使用localStorage存储项目数据，确保刷新后数据不丢失
- **导入导出**: 支持JSON、Markdown和Excel格式的数据导入导出
- **实时更新**: 任务操作后自动更新甘特图和WBS编号

## 3. 数据结构设计

### 3.1 项目信息

```javascript
{
  id: String,           // 项目ID
  name: String,         // 项目名称
  startDate: String,    // 项目开始日期
  endDate: String,      // 项目结束日期
  description: String,  // 备注
  members: [            // 项目人员
    {
      id: String,       // 人员ID
      name: String,     // 人员名字
      phone: String,    // 电话
      email: String,    // 邮箱
      role: String      // 角色
    }
  ]
}
```

### 3.2 任务信息

```javascript
{
  id: String,           // 任务ID
  wbs: String,          // WBS编号
  name: String,         // 任务名称
  startDate: String,    // 开始时间
  endDate: String,      // 结束时间
  duration: Number,     // 任务工期（天）
  deliverable: String,  // 交付物
  dependencies: [String], // 任务依赖
  assignee: String,     // 负责人
  priority: String,     // 任务优先级（高/中/低）
  status: String,       // 任务状态（待办/进行中/已完成）
  description: String,  // 任务备注
  parentId: String,     // 父任务ID
  children: [Object]    // 子任务
}
```

## 4. 功能模块设计

### 4.1 项目信息管理模块

- **功能**: 管理项目基本信息（项目名称、开始日期、结束日期、备注）和项目人员（姓名、电话、邮箱、角色）
- **组件**: ProjectInfoView, ProjectInfoForm, MemberManager
- **API**: 本地存储操作，支持JSON和Excel格式的导入导出

### 4.2 项目计划编制模块

- **功能**: 新增任务、新增子任务、编辑任务、删除任务、调整任务层级、调整任务顺序
- **组件**: ProjectPlanView, TaskList, TaskForm, TaskItem
- **API**: 本地存储操作，WBS编号生成，任务层级和顺序调整

### 4.3 甘特图模块

- **功能**: 生成甘特图，展示任务进度和依赖关系，支持拖拽调整任务时间和工期
- **组件**: GanttChart
- **API**: dhtmlx-gantt库操作，支持甘特图导出为PNG格式

### 4.4 导入导出模块

- **功能**: 导入导出项目信息和项目计划，支持JSON、Markdown和Excel格式
- **组件**: ImportExport
- **API**: 文件操作，Excel处理，Markdown生成

### 4.5 显示设置模块

- **功能**: 设置任务列表显示的字段（如WBS编号、任务名称、开始时间、结束时间、工期、交付物、依赖、负责人、优先级、状态、备注）
- **组件**: DisplaySettings
- **API**: 本地存储操作，用户偏好设置

## 5. 界面设计规范

### 5.1 布局规范

- **整体布局**: 顶部导航栏 + 内容区域 + 底部版权信息
- **内容区域**: 响应式设计，适配不同屏幕尺寸
- **甘特图**: 右侧自适应宽度，支持横向滚动

### 5.2 颜色规范

- **主色调**: #4285F4 (蓝色)
- **辅助色**: #34A853 (绿色), #FBBC05 (黄色), #EA4335 (红色)
- **中性色**: #F8F9FA (背景), #333333 (文本), #666666 (次要文本)

### 5.3 字体规范

- **标题**: 16-20px, 粗体
- **正文**: 14px, 常规
- **次要文本**: 12px, 常规

## 6. 导入导出功能实现

### 6.1 JSON格式

- **导出**: 将项目信息和任务数据转换为JSON字符串，下载为.json文件
- **导入**: 读取.json文件，解析为项目信息和任务数据

### 6.2 Excel格式

- **导出**: 使用xlsx库将数据转换为Excel格式，下载为.xlsx文件
- **导入**: 读取.xlsx文件，解析为项目信息和任务数据

## 7. 甘特图实现

### 7.1 核心功能

- **任务展示**: 以条形图形式展示任务的开始时间、结束时间和持续时间
- **依赖关系**: 以箭头形式展示任务之间的依赖关系
- **任务层级**: 根据任务层级缩进显示
- **交互功能**: 支持拖拽调整任务时间，点击查看任务详情

### 7.2 导出功能

- **PNG格式**: 将甘特图转换为图片，下载为.png文件

## 8. 性能优化

### 8.1 渲染优化

- **虚拟列表**: 当任务数量较多时，使用虚拟列表减少DOM节点
- **懒加载**: 甘特图组件懒加载，减少初始加载时间

### 8.2 数据处理优化

- **防抖处理**: 对频繁操作（如任务拖拽）进行防抖处理
- **批量更新**: 对批量操作（如任务排序）进行批量更新，减少渲染次数

## 9. 兼容性考虑

### 9.1 浏览器兼容性

- 支持主流浏览器：Chrome, Firefox, Safari, Edge
- 最低支持IE11

### 9.2 响应式设计

- 支持桌面端、平板端和移动端
- 在小屏幕设备上，任务列表和甘特图将垂直排列

## 10. 安全考虑

### 10.1 数据安全

- 本地存储数据加密
- 导入文件验证，防止恶意文件

### 10.2 操作安全

- 重要操作（如删除任务）添加确认机制
- 数据修改后自动保存，防止数据丢失
